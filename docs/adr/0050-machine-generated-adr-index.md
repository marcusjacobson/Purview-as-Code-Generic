# 0050 — ADR index is machine-generated; each ADR's three header lines are its only inputs

- **Status:** Accepted
- **Date:** 2026-07-13
- **Gates:** Cross-cutting; no [`project-plan.md`](../project-plan.md) §5 / §8 row. Gates the follow-up generator-fix item: repair the two [`docs-regen.yml`](../../.github/workflows/docs-regen.yml) parsing defects recorded below, migrate the curated prose in the [ADR 0044](0044-currency-watch-loops.md) and [ADR 0045](0045-template-kickoff-spinoff-model.md) index rows into those ADRs' own `Gates:` lines, correct [`README.md`](README.md) process step 4, and land the first clean regeneration of the "Current ADRs" table.
- **Deciders:** @marcusjacobson

## Context

The "Current ADRs" table in [`docs/adr/README.md`](README.md) is emitted by the `adr-index` step of [`.github/workflows/docs-regen.yml`](../../.github/workflows/docs-regen.yml), which walks `docs/adr/NNNN-*.md` and builds one row per ADR from exactly three inputs per file: the `# NNNN — Title` H1, the `- **Status:**` line, and the `- **Gates:**` line. Nothing else in an ADR reaches the table.

In practice the table has also been hand-edited on merge, because [`README.md`](README.md) process step 4 tells authors to "On merge, update the table above in the same PR." That instruction directly contradicts [`.github/skills/docs-maintenance/SKILL.md`](../../.github/skills/docs-maintenance/SKILL.md), which lists `docs/adr/README.md` and `docs/scripts-reference.md` as the mechanical indexes regenerated by `docs-regen.yml` and says "do not hand-edit those two files." The repo has been running both processes at once, and the hand-curated one has silently won every time. This ADR resolves the contradiction.

**Evidence gathered 2026-07-13.** A dry run of the generator against the committed table established:

1. **The table has never been generator-clean.** **0 of the 45** committed rows matched the generator's output byte-for-byte.
2. **The curated `Title` cells are editorial rewrites that do not exist in the ADR sources.** [ADR 0008](0008-portal-role-group-api.md)'s H1 reads "hybrid, Microsoft Graph primary, Security & Compliance PowerShell fallback", while its curated row reads "hybrid (Graph primary, S&C PowerShell fallback)". No mechanical formatting rule derives the second string from the first, so *reproducing the curated shape from the sources is impossible* — the curated titles can only be preserved by continuing to hand-write them.
3. **The curated `Gates` cells are hand-compressions of richer source lines.** The generator emits the full `Gates:` line from each ADR; the curated cells are shortened paraphrases of it. Regenerating therefore *gains* fidelity in the large majority of rows rather than losing it.
4. **Hand-curation has already produced a factual error.** [ADR 0049](0049-data-plane-sp-key-vault-firewall-rbac.md)'s `Gates:` line enumerates **three** `sync-*-from-tenant` workflows (`sync-labels-from-tenant`, `sync-label-policies-from-tenant`, `sync-auto-label-policies-from-tenant`), but the curated row in the table compresses them to "the `sync-*-from-tenant` **pair**". The index is now wrong about the repository it indexes — which is the failure mode a derived artifact is supposed to make impossible.

A derived index that is maintained by hand is not an index; it is a second, unvalidated source of truth that drifts from the first. Microsoft's Well-Architected Framework makes the general case for removing the human from mechanical, repeatable work, and the ADR table is precisely such work.

## Decision

We will treat the "Current ADRs" table in [`docs/adr/README.md`](README.md) as a **machine-generated artifact whose sole inputs are the ADR files themselves**. Specifically:

1. **The table is generated, never hand-edited.** [`.github/workflows/docs-regen.yml`](../../.github/workflows/docs-regen.yml) is the only writer of the "Current ADRs" table. No PR — including the PR that lands a new ADR — hand-adds, hand-edits, or hand-removes a row. This ADR is itself the first artifact governed by the rule: it does **not** add its own row to the table, and it does not need to. The generator will add row `0050` on its next run.
2. **Three header lines are the single source of truth for a row.** For every ADR file, the `# NNNN — Title` H1, the `- **Status:**` line, and the `- **Gates:**` line are the generator's only inputs, and therefore the single source of truth for that ADR's row. They **must be correct on merge of any ADR** — an ADR whose header lines are wrong ships a wrong index row, and no downstream edit can fix it without violating rule 1. Reviewers check the three header lines as a merge gate.
3. **Curated and editorial rewrites in the table are abandoned.** The shortened titles and compressed gates text currently in the table are not recoverable from the sources and are given up deliberately. Where a curated cell says something genuinely worth keeping, that prose is migrated *into the source ADR's header line* (see Consequences), not preserved in the table.

An ADR's row is a *view* of the ADR. If the view is wrong, the fix belongs in the ADR.

## Consequences

**The generator becomes authoritative, and the table becomes trustworthy.** The index can no longer disagree with the ADRs it indexes, because it is computed from them. The [ADR 0049](0049-data-plane-sp-key-vault-firewall-rbac.md) "pair" vs. three-workflows error is fixed by regeneration rather than by a second hand-edit that could be wrong in a new way. The curated titles are lost; that cost is accepted, since preserving them is exactly what requires the hand-editing this ADR abolishes.

**Header lines become a merge gate.** Authors and reviewers of any ADR now carry a load they did not carry before: the H1, `Status:`, and `Gates:` lines must be right *at merge time*, in the ADR itself. The H1 must use the em dash (`—`) separator, since that is the dominant convention and the form the generator parses.

**Two ADRs need their curated prose migrated before the first clean regeneration.** Exactly two rows — [ADR 0044](0044-currency-watch-loops.md) and [ADR 0045](0045-template-kickoff-spinoff-model.md) — carry curated `Gates` prose that has no counterpart in their source `Gates:` lines (0044's row names `watch-list.yml` and its fired trigger #1; 0045's row names `KickoffGuard.psm1`, `Set-KickoffGuard.ps1`, `Test-KickoffGuard.ps1`, and the `@operator-kickoff` agent). That prose must be migrated **into those two ADRs' `Gates:` lines** so that the first clean regeneration is lossless.

> **This is not a decision reversal.** Amending a `Gates:` header line on an already-`Accepted` ADR is **metadata maintenance**, not a re-opening of the decision: the `Gates:` line is a pointer to the work an ADR unblocks, not part of the decision's substance, and the ADR body — where the decision lives — is untouched. ADR immutability governs the *decision*, not its index metadata. The repo already has precedent for post-acceptance header edits: [ADR 0008](0008-portal-role-group-api.md)'s header carries a supersession note added on 2026-04-19, after acceptance, with the body "preserved unchanged for history."

**The docs-maintenance skill is correct and stands; the README process step is wrong and must be corrected.** [`.github/skills/docs-maintenance/SKILL.md`](../../.github/skills/docs-maintenance/SKILL.md) ("the mechanical indexes ... are regenerated separately by `docs-regen.yml`; do not hand-edit those two files") is ratified by this ADR and needs no change. [`docs/adr/README.md`](README.md) process step 4 ("On merge, update the table above in the same PR") contradicts it, is the proximate cause of the drift documented above, and must be corrected to state that the table is regenerated and that the three header lines must be correct on merge.

**Two generator defects must be fixed before the first clean regeneration can be trusted.** The generator is authoritative under this ADR, so its output must be right. Two parsing defects are known:

1. **Status truncation.** The status regex at [`docs-regen.yml:86`](../../.github/workflows/docs-regen.yml) —
   `(?:^|\*\*)\s*Status:\*\*\s+(.+?)(?:\s|<!--)` —
   uses a lazy quantifier terminated by a *single whitespace character*, so it stops at the first space in the status value. A status of `Superseded by` followed by a link to ADR 0009 collapses to a bare `Superseded`, losing the superseding ADR number. This affects [ADR 0008](0008-portal-role-group-api.md) and [ADR 0037](0037-unified-catalog-authoring-surface.md), the two superseded ADRs in the set.
2. **Title number leak.** The title-strip regex at [`docs-regen.yml:80`](../../.github/workflows/docs-regen.yml) —
   `^\d{4}\s+—\s+` —
   matches only the em dash. [ADR 0039](0039-irm-entity-list-tracked-fields.md) uses a single hyphen (`0039 - ...`) and [ADR 0042](0042-label-policy-admin-units.md) a double hyphen (`0042 -- ...`), so neither prefix is stripped and both leak the ADR number into the `Title` column, which already carries the number in its own linked cell.

Both defects, the 0044/0045 migration, the README step-4 correction, and the first clean regeneration are the scope of the follow-up item this ADR gates. They are deliberately **not** done here: this ADR *records* the decision, and shipping it separately keeps the ruling reviewable on its own.

**Security posture is unchanged.** This is a documentation-generation governance decision. It touches no identity, no secret, no tenant surface, and neither upholds nor relaxes any principle in [`security.instructions.md`](../../.github/instructions/security.instructions.md). It mildly *supports* the repo's provenance posture: a generated index is auditable from its sources, whereas a hand-edited one is not.

## Alternatives considered

**Alternative A: Do nothing — keep hand-curating the table.** Reject. This is the status quo, and the status quo is measurably broken: 0 of 45 rows are generator-clean, the two processes contradict each other in writing, and hand-curation has already published a factual error about [ADR 0049](0049-data-plane-sp-key-vault-firewall-rbac.md). Keeping it also means keeping a rule that no tooling can enforce and every author must remember.

**Alternative B: Make the generator reproduce the curated table.** Reject. It is not merely hard, it is impossible: the curated `Title` cells are editorial rewrites of the ADR H1s ([ADR 0008](0008-portal-role-group-api.md) being the clearest case), and no formatting rule derives them from the source. Encoding them would require a hand-maintained override map — i.e., reinventing hand-curation with extra machinery.

**Alternative C: Delete the table and link the ADR directory instead.** Reject. The table's `Status` and `Gates` columns are the reason it is read at all — they answer "is this ADR still live, and what does it unblock?" without opening 45 files. A bare directory listing answers neither. The problem is the table's *provenance*, not the table.

**Alternative D: Reverse the contradiction the other way — declare the table hand-owned and delete the generator.** Reject. It trades an enforceable, auditable source of truth for one that depends on every future author remembering a manual step, and the evidence shows exactly how that ends. It would also strand [`docs/scripts-reference.md`](../scripts-reference.md), which is generated by the same workflow on the same principle.
